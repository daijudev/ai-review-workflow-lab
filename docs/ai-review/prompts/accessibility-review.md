# アクセシビリティレビュープロンプト

フロントエンドのアクセシビリティ（a11y）を確認するためのプロンプトです。
WCAG 2.1 AA 準拠を目標とするレビューに使用してください。

---

## プロンプト

```
あなたはWebアクセシビリティの専門家です。
以下のReactコンポーネントのアクセシビリティをレビューしてください。
WCAG 2.1 AA 準拠を基準としてください。

## レビュー観点

### 1. セマンティックHTML

- [ ] 適切なHTML要素を使っているか（ボタンは `<button>`、リンクは `<a>` 等）
- [ ] 見出し階層が論理的か（h1 → h2 → h3 の順序）
- [ ] リストは `<ul>` / `<ol>` / `<li>` で適切にマークアップされているか
- [ ] `<div>` / `<span>` のクリックイベントに `role` と `tabIndex` が設定されているか

### 2. フォームのアクセシビリティ

- [ ] すべての入力要素に `<label>` が関連付けられているか（`for` 属性または `aria-label`）
- [ ] 必須項目が `required` 属性と視覚的な表示の両方で示されているか
- [ ] バリデーションエラーが入力要素と関連付けられているか（`aria-describedby`）
- [ ] エラーメッセージが `role="alert"` または `aria-live="polite"` で通知されるか
- [ ] プレースホルダーをラベルの代わりに使っていないか

### 3. キーボードナビゲーション

- [ ] すべてのインタラクティブ要素がキーボードで操作できるか
- [ ] フォーカス順序が論理的か（視覚的な順序と一致しているか）
- [ ] フォーカスが視覚的に分かるか（フォーカスリングが非表示になっていないか）
- [ ] モーダル・ドロップダウンでのフォーカストラップが実装されているか
- [ ] `Escape` キーでモーダルを閉じられるか

### 4. ARIA属性

- [ ] `aria-label` / `aria-labelledby` がアイコンボタンやリージョンに設定されているか
- [ ] `aria-expanded` がアコーディオン・ドロップダウンに設定されているか
- [ ] `aria-selected` / `aria-checked` / `aria-pressed` が適切に使われているか
- [ ] `aria-disabled` と `disabled` 属性が混在していないか（どちらかに統一する）
- [ ] `role` 属性は必要な場合のみ使っているか（ネイティブHTML要素で代替できないか）

### 5. 画像・アイコン

- [ ] 意味のある画像に `alt` テキストが設定されているか
- [ ] 装飾目的の画像は `alt=""` になっているか
- [ ] アイコンのみのボタン・リンクに `aria-label` が設定されているか
- [ ] SVG アイコンが `aria-hidden="true"` になっているか（テキストラベルがある場合）

### 6. カラーとコントラスト

- [ ] テキストのコントラスト比が WCAG AA 基準を満たしているか（通常テキスト: 4.5:1以上）
- [ ] 情報をカラーのみで伝えていないか（アイコン・テキスト・パターンを併用する）
- [ ] フォーカス時のコントラスト比が十分か

### 7. 動的コンテンツ

- [ ] ページ遷移・コンテンツ更新時にスクリーンリーダーに通知されるか
- [ ] loading 状態が `aria-busy="true"` または `aria-live` で通知されるか
- [ ] toast / snackbar 等の通知が `role="alert"` / `aria-live="polite"` で実装されているか

## 補足

- 確信がない指摘には「要確認:」と前置きをつけてください
```

## 簡易チェックリストとして使う場合

特定の機能（フォーム・モーダル・ナビゲーション等）に絞って確認したい場合は、
以下のように観点を絞ってください。

```
以下のフォームコンポーネントのアクセシビリティを「フォームのアクセシビリティ」の観点のみでレビューしてください。

[コードを貼り付け]
```
